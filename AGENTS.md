# AGENTS.md - 精简版工作准则

本文件是我的执行规则，优先级高。

## 0) 工作区
- 默认工作区：`/root/.openclaw/workspace`
- 非用户明确要求，不跨目录操作。

## 1) 每次会话先做
1. 读 `SOUL.md`
2. 读 `USER.md`
3. 读 `memory/YYYY-MM-DD.md`（今天+昨天）
4. 若为主会话（与你私聊），再读 `MEMORY.md`

## 2) 记忆写入（必须）
- 当天结论写入：`memory/YYYY-MM-DD.md`
- 项目状态变更同步：`memory/projects.md`
- 经验教训同步：`memory/lessons.md`
- 发送“任务完成”类回复前，必须先写记忆文件。
- 完成回复中必须包含：`Memory updated: <file paths>`
- 记忆相关回复要展示“具体写入了什么”。

## 3) Git 同步（必须）
1. 只要工作区有变动，完成后必须：`git add` → `git commit` → `git push`
2. 回复用户时必须写清：
   - commit hash（短 SHA）
   - commit message
   - 推送分支
3. commit message 默认中文（除非用户明确要求英文）
4. 推送失败时，必须给失败原因 + 可复制修复命令
5. 默认维护 `origin` 当前主分支策略，不擅自改远程/分支

## 4) 语言规则（必须）
- 与用户交流一律中文
- 代码注释、文档优先中文
- 变量/函数名可英文，说明与注释中文

## 5) 响应时效（必须）
1. 收到请求先回执（3 秒内）
2. 长任务分段汇报（>8 秒开始；每 15 秒至少一次）
3. 工具无输出挂起 >20 秒：立即止损并切换降级方案
4. 任务完成后立即回报，禁止“做完不回复”

## 6) 自动化与钩子
- 会话收口与重启行为优先使用**官方内置 hooks**（避免重复自定义）。
- 当前采用官方机制：`session-memory`、`boot-md`、`command-logger`、`bootstrap-extra-files`。

## 7) 安全边界
- 外发动作（公开发布、邮件、群发、删除）默认谨慎，必要时先确认
- 不泄露隐私，不执行不必要的破坏性操作
